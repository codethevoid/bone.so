## Examples

The following variants are available for the Button component:

### Primary

Primary buttons are the default style and are used for main actions.

<Showcase>
  <Button>Primary</Button>
</Showcase>

<CodeBlock filename={"primary-btn.tsx"} code={primary} />

### Soft

Soft buttons have a subtle background and are used for less prominent actions.

<Showcase>
  <Button variant="soft">Soft</Button>
</Showcase>

<CodeBlock filename={"soft-btn.tsx"} code={soft} />

### Outline

Outline buttons have a border and are used for secondary actions.

<Showcase>
  <Button variant="outline">Outline</Button>
</Showcase>

<CodeBlock filename={"outline-btn.tsx"} code={outline} />

### Ghost

Ghost buttons have no background or border and are used for minimalistic designs.

<Showcase>
  <Button variant="ghost">Ghost</Button>
</Showcase>

<CodeBlock filename={"ghost-btn.tsx"} code={ghost} />

### Danger

Danger buttons are used for destructive actions and have a red color scheme.

<Showcase>
  <Button variant="danger">Danger</Button>
</Showcase>

<CodeBlock filename={"danger-btn.tsx"} code={danger} />

### Custom

You can apply custom styles using Tailwind CSS classes.

<Showcase>
  <Button
    className="border-0 bg-gradient-to-tr from-blue-400 to-blue-600 text-white shadow-xl shadow-blue-600/20 hover:scale-[103%] hover:opacity-100 focus:ring-0 active:scale-[98%] dark:shadow-blue-500/20">
    <span>Custom</span>
  </Button>
</Showcase>

<CodeBlock filename="custom-btn.tsx" code={custom} />

### Link

You can make any element look like the button component by using the `passThrough` prop. This is useful for creating links that look like buttons.

<Showcase>
  <Button passThrough>
    <a href="https://github.com/codethevoid/bone.so" target="_blank" rel="noopener noreferrer">
      <span>View Github</span>
    </a>
  </Button>
</Showcase>

<CodeBlock filename="link-btn.tsx" code={link} />
